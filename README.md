# 로또 프로그램

## 프로젝트 설명
이 프로젝트는 간단한 로또 발매기를 구현하는 자바 프로그램입니다. 사용자가 로또 구입 금액을 입력하면 구입 금액에 해당하는 수량의 로또를 발행하고, 당첨 번호와 비교하여 당첨 내역 및 수익률을 계산합니다.

## 기능 설명

### 1️⃣ 구입 금액 입력받기
- [x] 사용자로부터 구입 금액을 입력받음

#### 예외 처리
- 구입 금액이 1000원 단위가 아닐 경우
- 구입 금액이 0이거나 음수인 경우
- 구입 금액이 문자열인 경우

#### 구매 수량 계산
```
구매 수량 = 구입 금액 / 1000
```


### 2️⃣ 로또 발행하기
- [x] 구매 수량만큼 로또를 발행
- [x] 로또 번호는 오름차순으로 정렬
- [x] 발행한 로또 번호를 1개씩 모두 출력


### 3️⃣ 당첨 번호 입력받기
- [x] 당첨 번호를 쉼표로 구분하여 입력받음

#### 예외 처리
- 모든 번호는 1~45 사이에 포함해야 한다
- 번호는 중복되지 않아야 한다
- 번호는 6개여야 한다
- 당첨 번호가 문자열인 경우

### 4️⃣ 보너스 번호 입력받기
- [x] 보너스 번호를 1개 입력받는다

#### 예외 처리
- 보너스 번호가 1개가 아닌 경우
- 보너스 번호가 문자인 경우
- 보너스 번호는 1~45 사이에 포함되어야 한다
- 보너스 번호는 당첨 번호와 일치하면 안된다


### 5️⃣ 로또 번호와 당첨 번호 비교하기
- [x] 각각의 로또 번호와 당첨 번호를 비교하여 당첨 개수를 센다

#### 당첨 조건
- 당첨 번호가 2개 이하 포함되어 있으면 당첨 X
- 당첨 번호가 3개 포함되어 있으면 **5등**
- 당첨 번호가 4개 포함되어 있으면 **4등**
- 당첨 번호가 5개 포함되고, 보너스 번호도 포함되면 **2등**
- 당첨 번호가 5개 포함되고, 보너스 번호가 포함되지 않으면 **3등**
- 당첨 번호가 6개 포함되어 있으면 **1등**

### 6️⃣ 당첨 통계 출력하기
- [x] 당첨 내역과 총 수익률을 출력한다

#### 출력 내용
- **당첨 내역** : 5등부터 1등까지 등수별 당첨 개수를 출력한다
- **총 수익률** 
    ```
    총 수익금 = (당첨 개수 * 당첨 금액)의 합
    총 수익률 = (총 수익금 / 구입 금액) * 100
    총 수익률은 소수점 둘째 자리에서 반올림한다
  ```
    

### 예외 처리
- 입력된 값이 잘못된 경우 `IllegalArgumentException`을 발생시키고, `[ERROR]`로 시작하는 에러 메시지를 출력 후 다시 입력받습니다.
- 예외 상황 예시:
    - 구입 금액이 1,000원 단위가 아닐 경우
    - 로또 번호가 1~45 범위를 벗어날 경우

### 입력 예시
1. **로또 구입 금액**:
    - 구입 금액을 입력받습니다. 구입 금액은 1,000원 단위로 입력받으며, 1,000원으로 나누어 떨어지지 않는 경우 예외 처리합니다.
    - 예시 입력:
      ```
      14000
      ```

2. **당첨 번호**:
    - 당첨 번호를 쉼표(,)로 구분하여 입력받습니다.
    - 예시 입력:
      ```
      1,2,3,4,5,6
      ```

3. **보너스 번호**:
    - 보너스 번호를 입력받습니다.
    - 예시 입력:
      ```
      7
      ```

### 출력 예시
1. **발행한 로또 수량 및 번호**:
    - 로또 번호는 오름차순으로 정렬하여 출력합니다.
    - 예시 출력:
      ```
      8개를 구매했습니다.
      [8, 21, 23, 41, 42, 43] 
      [3, 5, 11, 16, 32, 38] 
      [7, 11, 16, 35, 36, 44] 
      [1, 8, 11, 31, 41, 42] 
      [13, 14, 16, 38, 42, 45] 
      [7, 11, 30, 40, 42, 43] 
      [2, 13, 22, 32, 38, 45] 
      [1, 3, 5, 14, 22, 45]
      ```

2. **당첨 내역**:
    - 각 등수에 해당하는 당첨 횟수를 출력합니다.
    - 예시 출력:
      ```
      당첨 통계
      ---
      3개 일치 (5,000원) - 1개
      4개 일치 (50,000원) - 0개
      5개 일치 (1,500,000원) - 0개
      5개 일치, 보너스 볼 일치 (30,000,000원) - 0개
      6개 일치 (2,000,000,000원) - 0개
      ```

3. **수익률**:
    - 수익률을 소수점 둘째 자리에서 반올림하여 출력합니다.
    - 예시 출력:
      ```
      총 수익률은 62.5%입니다.
      ```

### 예외 메시지
- 잘못된 값이 입력된 경우 에러 메시지를 출력합니다. 에러 메시지는 `[ERROR]`로 시작합니다.
- 예시 에러 메시지: 
    ```
  [ERROR] 로또 번호는 1부터 45 사이의 숫자여야 합니다.
    ```

## 프로그래밍 요구 사항

### 프로그래밍 요구 사항 1
- JDK 21 버전에서 실행 가능해야 한다.
- 프로그램 실행의 시작점은 `Application`의 `main()`이다.
- `build.gradle` 파일은 변경할 수 없으며, 제공된 라이브러리 이외의 외부 라이브러리는 사용하지 않는다.
- 프로그램 종료 시 `System.exit()`를 호출하지 않는다.
- 프로그래밍 요구 사항에서 달리 명시하지 않는 한 파일, 패키지 등의 이름을 바꾸거나 이동하지 않는다.
- 자바 코드 컨벤션을 지키면서 프로그래밍한다.
- 기본적으로 Java Style Guide를 원칙으로 한다.

### 프로그래밍 요구 사항 2
- `indent`(인덴트, 들여쓰기) depth를 3이 넘지 않도록 구현한다. **2까지만 허용**한다.
    - 예를 들어 `while`문 안에 `if`문이 있으면 들여쓰기는 2이다.
    - 힌트: indent(인덴트, 들여쓰기) depth를 줄이는 좋은 방법은 **함수(또는 메서드)를 분리**하면 된다.
- **3항 연산자를 쓰지 않는다.**
- 함수(또는 메서드)가 한 가지 일만 하도록 최대한 작게 만들어라.
- **JUnit 5**와 **AssertJ**를 이용하여 정리한 기능 목록이 정상적으로 작동하는지 테스트 코드로 확인한다.
    - 테스트 도구 사용법이 익숙하지 않다면 아래 문서를 참고하여 학습한 후 테스트를 구현한다.
    - [JUnit 5 User Guide](https://junit.org/junit5/docs/current/user-guide/)
    - [AssertJ User Guide](https://assertj.github.io/doc/)
    - [AssertJ Exception Assertions](https://assertj.github.io/doc/#assertj-core-exception-assertions)
    - [Guide to JUnit 5 Parameterized Tests](https://www.baeldung.com/parameterized-tests-junit-5)

### 프로그래밍 요구 사항 3
- 함수(또는 메서드)의 길이가 **15라인을 넘어가지 않도록** 구현한다.
- 함수(또는 메서드)가 한 가지 일만 잘 하도록 구현한다.
- `else` 예약어를 쓰지 않는다.
    - `else`를 쓰지 말라고 하니 `switch/case`로 구현하는 경우가 있는데 `switch/case`도 허용하지 않는다.
    - 힌트: `if` 조건절에서 값을 `return`하는 방식으로 구현하면 `else`를 사용하지 않아도 된다.
- Java `Enum`을 적용하여 프로그램을 구현한다.
- 구현한 기능에 대한 단위 테스트를 작성한다. 단, **UI(`System.out`, `System.in`, `Scanner`) 로직은 제외**한다.
    - 단위 테스트 작성이 익숙하지 않다면 `LottoTest`를 참고하여 학습한 후 테스트를 작성한다.

## 라이브러리
- `camp.nextstep.edu.missionutils`에서 제공하는 `Randoms` 및 `Console` API를 사용하여 구현해야 한다.
    - **Random 값 추출**은 `camp.nextstep.edu.missionutils.Randoms`의 `pickUniqueNumbersInRange()`를 활용한다.
    - **사용자가 입력하는 값**은 `camp.nextstep.edu.missionutils.Console`의 `readLine()`을 활용한다.
    - **사용 예시**

          ```
          Randoms.pickUniqueNumbersInRange(1, 45, 6);
          ```

## Lotto 클래스
- 제공된 `Lotto` 클래스를 사용하여 구현해야 한다.
- `Lotto`에 `numbers` 이외의 필드(인스턴스 변수)를 추가할 수 없다.
- `numbers`의 접근 제어자인 `private`은 변경할 수 없다.
- `Lotto`의 패키지를 변경할 수 있다.

```
public class Lotto {
    private final List<Integer> numbers;

    public Lotto(List<Integer> numbers) {
        validate(numbers);
        this.numbers = numbers;

    private void validate(List<Integer> numbers) {
        if (numbers.size() != 6) {
            throw new IllegalArgumentException("[ERROR] 로또 번호는 6개여야 합니다.");
        }
```
