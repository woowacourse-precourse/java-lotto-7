# java-lotto-precourse 기능 정의서

----

## [ 0. 목차 ]
### 1.사용자 입력
### 2. 로또 번호 발행기
### 3. 로또 당첨 확인기
### 4. 수익률 계산기
### 5. 사용자 출력
### 6. 발생 가능 예외 사항

----

## [ 1. 사용자 입력 ]

### ✅ 1.1. 로또 구입 입력 금액을 입력받는다.
#### 1.1.1 구입 금액은 1,000원 단위이다.
#### 1.1.2 1,000원으로 나누어 떨어지지 않는 경우 예외 처리 한다.

### ✅ 1.2. 사용자 예측 당첨 번호를 입력 받는다.
#### 1.2.1. 번호는 쉼표(,)를 기준으로 구분한다.
#### 1.2.2. 번호는 1 이상이어야 한다.

### ✅ 1.3. 보너스 번호를 입력 받는다.
#### 1.3.1. 보너스 번호는 1 이상의 수 하나만 입력받는다.

----

## [ 2. 로또 번호 발행기 ]

### ✅ 2.1 로또 번호를 발행한다.
#### 2.1.1 로또 번호의 숫자 범위는 1-45 까지이다.
#### 2.1.2 당첨 번호 추첨 시 중복되지 않는 숫자 6개와 보너스 번호 1개를 뽑는다.
#### 2.1.3. 입력값은 사용자가 입력한 구입 금액이다.
#### 2.1.4. 구입 금액(1,000원 단위)에 해당하는 만큼 로또를 발행한다.

----

## [ 3. 로또 당첨 확인기 ]

### ✅ 3.1 로또의 당첨을 확인한다.
#### 3.1.1. 1등: 6개 번호 일치 / 2,000,000,000원 
#### 3.1.2. 2등: 5개 번호 + 보너스 번호 일치 / 30,000,000원
#### 3.1.3. 3등: 5개 번호 일치 / 1,500,000원 
#### 3.1.4. 4등: 4개 번호 일치 / 50,000원
#### 3.1.5. 5등: 3개 번호 일치 / 5,000원
#### 3.1.6. 그 외 : 0개 이상 2개 이하 번호 일치 / 0원

----

## [ 4. 수익률 계산기 ]

### ✅ 4.1 수익률을 계산한다.
#### 4.1.1. 수익률은 소수점 둘째 자리에서 반올림

----

## [ 5. 사용자 출력 ]

### ✅ 5.1. 사용자로부터 구입금액 입력받을 수 있도록 안내 문구를 출력한다.
#### 5.1.1. 출력형식 : 구입금액을 입력해주세요.

### ✅ 5.2. 발행한 로또 수량 및 번호를 출력한다.
#### 5.2.1. 출력형식 : n개를 구매했습니다.
#### 5.2.2. 발행된 로또는 대괄호로 묶여있고, 쉼표(,)로 구분되어 있다.
#### 5.2.3. 로또 번호는 오름차순으로 정렬하여 보여준다.

### ✅ 5.3. 사용자로부터 예측 로또 당첨 번호 입력 안내 문구를 출력한다.
#### 5.3.1. 출력형식 : 당첨 번호를 입력해 주세요.

### ✅ 5.4. 사용자로부터 보너스 번호를 입력받을 수 있도록 안내 문구를 출력한다.
#### 5.3.1. 출력형식 : 보너스 번호를 입력해 주세요.

### ✅ 5.5 당첨 내역을 출력한다.
#### 5.5.1. 출력형식 : 당첨 통계|n---
#### 5.5.2. 3개 일치 (5,000원) - n개
#### 5.5.3. 4개 일치 (50,000원) - n개
#### 5.5.4. 5개 일치 (1,500,000원) - n개
#### 5.5.5. 5개 일치, 보너스 볼 일치 (30,000,000원) - n개
#### 5.5.6. 6개 일치 (2,000,000,000원) - n개

### ✅ 5.6. 발생한 수익률을 출력한다.
#### 5.6.1. 출력형식 : 총 수익률은 nn.n%입니다.

### ✅ 5.7. 예외 상황 발생 시 에러 문구를 출력한다.
#### 5.7.1. 에러 문구는 [ERROR]로 시작해야 한다.

---

## [ 6. 발생 가능 예외 사항 ]

### ✅ 6.1. 사용자로부터 입력받는 금액이 1,000원 단위가 아닌 경우
#### 6.1.1. 예외 클래스: IllegalArgumentException
#### 6.1.2. 에러 메시지: [ERROR] 금액은 1,000원 단위로 입력해야 합니다.

### ✅ 6.2. 사용자로부터 입력받는 금액에 특수기호가 포함된 경우
#### 6.2.1. 예외 클래스: NumberFormatException
#### 6.2.2. 에러 메시지: [ERROR] 금액 입력에 특수 기호를 사용할 수 없습니다.

### ✅ 6.3. 사용자가 입력한 로또 번호가 1에서 45 범위에서 벗어난 경우
#### 6.3.1. 예외 클래스: IllegalArgumentException
#### 6.3.2. 에러 메시지: [ERROR] 로또 번호는 1에서 45 사이의 숫자여야 합니다.

### ✅ 6.4. 사용자가 입력한 로또 번호가 6개가 아닌 경우
#### 6.4.1. 예외 클래스: IllegalArgumentException
#### 6.4.2 에러 메시지: [ERROR] 로또 번호는 6개여야 합니다.

### ✅ 6.5. 사용자가 입력한 보너스 번호가 1에서 45 범위에서 벗어난 경우
#### 6.5.1. 예외 클래스: IllegalArgumentException
#### 6.5.2 에러 메시지: [ERROR] 보너스 번호는 1에서 45 사이의 숫자여야 합니다.

### ✅ 6.6. 사용자가 입력한 로또 번호가 숫자가 아닌 경우
#### 6.6.1. 예외 클래스: NumberFormatException
#### 6.6.2. 에러 메시지: [ERROR] 로또 번호는 숫자만 입력할 수 있습니다.

### ✅ 6.7. 사용자가 입력한 로또 번호의 구분자가 쉼표(,)가 아닌 경우
#### 6.7.1. 예외 클래스: IllegalArgumentException
#### 6.7.2. 에러 메시지: [ERROR] 로또 번호는 쉼표(,)로 구분해야 합니다.

### ✅ 6.8. 사용자가 입력한 보너스 번호가 1개가 아닌 경우
#### 6.8.1. 예외 클래스: IllegalArgumentException
#### 6.8.2. 에러 메시지: [ERROR] 보너스 번호는 1개여야 합니다.

### ✅ 6.9. 사용자가 입력한 보너스 번호가 숫자가 아닌 경우
#### 6.9.1. 예외 클래스: NumberFormatException
#### 6.9.2. 에러 메시지: [ERROR] 보너스 번호는 숫자만 입력할 수 있습니다.

### ✅ 6.10. 사용자가 입력한 보너스 번호가 숫자가 아닌 경우
#### 6.10.1. 예외 클래스: NumberFormatException
#### 6.10.2. 에러 메시지: [ERROR] 금액은 숫자만 입력할 수 있습니다.

### ✅ 6.11. 사용자가 입력한 로또 번호 및 보너스 번호가 겹치는 상황
#### 6.11.1. 예외 클래스: IllegalException
#### 6.11.2. 에러 메시지: [ERROR] "로또 번호는 중복되어선 안됩니다."
#### 6.11.3. 에러 메시지: [ERROR] "보너스 번호는 중복되어선 안됩니다."
