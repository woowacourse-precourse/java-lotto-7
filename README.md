## 🚀 요구사항

### ✔️ 기능 요구사항

간단한 로또 발매기를 구현한다.

+ 로또 번호의 숫자 범위는 1~45까지이다.
+ 1개의 로또를 발행할 때 중복되지 않는 6개의 숫자를 뽑는다.
+ 당첨 번호 추첨 시 중복되지 않는 숫자 6개와 보너스 번호 1개를 뽑는다.
+ 당첨은 1등부터 5등까지 있다. 당첨 기준과 금액은 아래와 같다. 
  + 1등: 6개 번호 일치 / 2,000,000,000원 
  + 2등: 5개 번호 + 보너스 번호 일치 / 30,000,000원 
  + 3등: 5개 번호 일치 / 1,500,000원 
  + 4등: 4개 번호 일치 / 50,000원 
  + 5등: 3개 번호 일치 / 5,000원
+ 로또 구입 금액을 입력하면 구입 금액에 해당하는 만큼 로또를 발행해야 한다. 
+ 로또 1장의 가격은 1,000원이다. 
+ 당첨 번호와 보너스 번호를 입력받는다. 
+ 사용자가 구매한 로또 번호와 당첨 번호를 비교하여 당첨 내역 및 수익률을 출력하고 로또 게임을 종료한다. 
+ 사용자가 잘못된 값을 입력할 경우 `IllegalArgumentException`을 발생시키고, "[ERROR]"로 시작하는 에러 메시지를 출력 후 그 부분부터 입력을 다시 받는다. 
  + Exception이 아닌 `IllegalArgumentException`, `IllegalStateException` 등과 같은 명확한 유형을 처리한다.

<br>

### ✔️ 입출력 요구사항

#### 입력 

+ 로또 구입 금액을 입력 받는다. 구입 금액은 1,000원 단위로 입력 받으며 1,000원으로 나누어 떨어지지 않는 경우 예외 처리한다.
  
    ``` 
    14000   
    ```

+ 당첨 번호를 입력 받는다. 번호는 쉼표(,)를 기준으로 구분한다.
    ```
    1,2,3,4,5,6
    ```

+ 보너스 번호를 입력 받는다.
    ```
    7
    ```

<br>

#### 출력

+ 발행한 로또 수량 및 번호를 출력한다. 로또 번호는 오름차순으로 정렬하여 보여준다.
    ```
    8개를 구매했습니다.
    [8, 21, 23, 41, 42, 43]
    [3, 5, 11, 16, 32, 38]
    [7, 11, 16, 35, 36, 44]
    [1, 8, 11, 31, 41, 42]
    [13, 14, 16, 38, 42, 45]
    [7, 11, 30, 40, 42, 43]
    [2, 13, 22, 32, 38, 45]
    [1, 3, 5, 14, 22, 45]
    ```

+ 당첨 내역을 출력한다.
    ```
    3개 일치 (5,000원) - 1개
    4개 일치 (50,000원) - 0개
    5개 일치 (1,500,000원) - 0개
    5개 일치, 보너스 볼 일치 (30,000,000원) - 0개
    6개 일치 (2,000,000,000원) - 0개
    ```

+ 수익률은 소수점 둘째 자리에서 반올림한다. (ex. 100.0%, 51.5%, 1,000,000.0%)
    ```
    총 수익률은 62.5%입니다.
    ```

+ 예외 상황 시 에러 문구를 출력해야 한다. 단, 에러 문구는 "[ERROR]"로 시작해야 한다.
    ```
    [ERROR] 로또 번호는 1부터 45 사이의 숫자여야 합니다.
    ```

<br>

## 🎯 기능 목록

### ✔️ 입력

- [x] 로또 구입 금액을 입력받는다.
  - [x] 구입 금액은 1000원 단위로 입력받는다.
  - [x] `예외상황` :  1000원 단위로 나누어 떨어지지 않으면 예외를 발생시킨다.
- [x] 당첨 번호를 입력 받는다. 번호는 쉼표(,)를 기준으로 구분한다.
  - [x] 당첨 번호는 [1,45] 사이의 숫자여야 한다.
  - [x] 당첨 번호는 6개다.
  - [x] 당첨 번호끼리 중복되지 않는다.
  - [x] `예외상황`: 숫자가 아닌 값이 포함되면 예외를 발생시킨다.
  - [x] `예외상황`: 당첨 번호가 6개를 초과하거나 6개 미만으로 입력하면 예외를 발생시킨다.
  - [x] `예외상황`: 당첨 번호에 [1,45] 범위를 벗어나는 숫자가 있을 경우 예외를 발생시킨다.
  - [x] `예외상황`: 당첨 번호끼리 중복되면 예외를 발생시킨다.
  
- [x] 보너스 번호를 입력 받는다.
  - [x] 보너스 번호는 [1,45] 사이의 숫자여야 한다.
  - [x] 보너스 번호는 1개여야 한다.
  - [x] 보너스 번호는 당첨번호와 중복되지 않는다.
  - [x] `예외상황`: 숫자가 아닌 값이 포함되면 예외를 발생시킨다.
  - [x] `예외상황`: 숫자가 1개가 아니면 예외를 발생시킨다.
  - [x] `예외상황`: 당첨 번호에 [1,45] 범위를 벗어나는 숫자가 있을 경우 예외를 발생시킨다.
  - [x] `예외상황`: 보너스 번호가 당첨 번호와 중복되면 예외를 발생시킨다.

<br>

### ✔️ 로또 티켓

- [x] 로또 티켓은 번호를 자동으로 생성한다. 
  - [x] 각 티켓은 1부터 45 사이의 중복되지 않는 6개의 숫자로 구성된다. 
  - [x] 티켓의 번호는 오름차순으로 정렬된다.

<br>

### ✔️ 로또 당첨 확인
- [x] 각 티켓의 번호와 당첨 번호를 비교하여 일치하는 번호 개수를 확인한다.
  - 1등: 6개 번호 일치
  - 2등: 5개 번호 + 보너스 번호 일치
  - 3등: 5개 번호 일치
  - 4등: 4개 번호 일치
  - 5등: 3개 번호 일치
- [x] 각 티켓의 등수를 기록하여 당첨 개수를 저장한다.

<br>

### ✔️ 총 상금 및 수익률 계산 
- [x] 각 등수에 해당하는 티켓 개수를 집계하여 총 상금을 계산한다.
  - 1등: 2,000,000,000원 
  - 2등: 30,000,000원 
  - 3등: 1,500,000원 
  - 4등: 50,000원 
  - 5등: 5,000원
- [x] 총 당첨 상금을 구입 금액과 비교하여 수익률을 계산한다. 
  - [x] 수익률은 `(총 당첨 상금 / 총 구입 금액) * 100`으로 계산하여 소수점 둘째 자리까지 반올림하여 출력한다.

<br>


### ✔️ 예외 처리

- [x] 사용자가 입력한 값이 유효하지 않을 때, 다음과 같은 예외 처리 및 재입력 절차를 구현한다.
  - [x] [ERROR] 메시지를 통해 잘못된 입력 내용을 보여준다.
  - [x] 잘못된 금액, 잘못된 당첨 번호 및 보너스 번호 입력 시 오류 메시지를 출력하고 재입력을 요구한다.


<br>

### ✔️ 출력
- [x] 발행한 로또 티켓 수와 각 티켓의 번호를 오름차순으로 출력한다. 
  - [x] 구입 금액에 따라 발행된 로또 티켓 수를 출력한다.
  - [x] 각 티켓의 번호는 중복 없이 오름차순으로 출력한다.
- [x] 당첨 내역을 출력한다. 
  - [x] 각 등수에 따른 당첨 개수와 당첨 상금을 출력한다.
  - [x] 총 당첨 상금, 구입 금액 대비 수익률을 함께 출력한다.

<br>
